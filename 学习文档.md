# 第一部分：硬件端 (ESP32) —— “耳朵与搬运工”

### 1. 原理讲解

ESP32 在这里扮演了两个角色：

1. **耳朵 (采集)**：通过 I2S 接口从 INMP441 麦克风读取声音信号。
2. **搬运工 (上传)**：通过 WiFi 将声音打包，利用 HTTP 协议发送给服务器。

#### 关键技术点：

- **I2S (Inter-IC Sound)**：这是一种专门传输数字音频数据的总线标准。相比于模拟信号（ADC），它的抗干扰能力强，音质好。
- **PCM 数据 vs WAV 格式**：
  - 麦克风录下来的是 **PCM (脉冲编码调制)** 数据，它是一堆纯粹的数字，没有任何说明书。
  - 电脑播放器不知道这些数字代表什么（采样率多少？单声道还是立体声？）。
  - 所以我们需要给它加一个 **WAV 文件头 (Header)**。这就好比给一箱散装的货物贴上快递单，写明“这是16kHz采样、16位深度的音频”，这样电脑才能播放。
- **PSRAM (伪静态随机存储器)**：音频数据非常大（10秒钟约320KB）。ESP32 内部内存很小，所以必须用到 ESP32-S3 N16R8 特有的 8MB 外部内存（PSRAM）来暂存这些数据。
- **HTTP Multipart Upload**：为了模拟像网页上“上传文件”一样的行为，我们需要手动构建 HTTP 协议的 `multipart/form-data` 格式。这就像是把音频文件装进一个特定格式的信封里寄出去。

### 2. 核心配置与代码对应

**文件：`ESP32/ESP32.ino`**

- **配置 I2S (耳朵)**： 告诉 ESP32 怎么听。

  C++

  ```
  // ================= I2S 初始化函数 =================
  // 作用：配置 ESP32 的 I2S 硬件接口，使其能够与麦克风和扬声器通信。
  // 原理：ESP32 有两个 I2S 端口 (0和1)，我们分别用于输入和输出。
  void i2s_install() {
     // -----------------------------------------------------------
    // 1. 配置 I2S_NUM_0 用于麦克风 (录音/输入)
    // -----------------------------------------------------------
    i2s_config_t i2s_config_mic = {
      .mode = (i2s_mode_t)(I2S_MODE_MASTER | I2S_MODE_RX), // 主机模式，接收数据
      .sample_rate = SAMPLE_RATE,
      .bits_per_sample = I2S_BITS_PER_SAMPLE_16BIT,        // 16位采样精度
      .channel_format = I2S_CHANNEL_FMT_ONLY_LEFT,         // 单声道 (左声道)
      .communication_format = I2S_COMM_FORMAT_I2S,         // 标准 I2S 格式
      .intr_alloc_flags = ESP_INTR_FLAG_LEVEL1,            //最低优先级
      .dma_buf_count = 8,                                  // DMA 缓冲区数量
      .dma_buf_len = 64,                                   // 每个缓冲区长度
      .use_apll = false                                  // 不使用 APLL (高精度音频锁相环)，使用普通时钟源以节省功耗
    };
    i2s_driver_install(I2S_NUM_0, &i2s_config_mic, 0, NULL);  // 安装驱动：根据上面的配置初始化 I2S_NUM_0 硬件
    
    // 引脚映射配置：将 I2S_NUM_0 的信号连接到具体的 GPIO 引脚
    i2s_pin_config_t pin_config_mic = {
      .bck_io_num = MIC_I2S_SCK,
      .ws_io_num = MIC_I2S_WS,
      .data_out_num = -1,            // 麦克风不需要输出引脚
      .data_in_num = MIC_I2S_SD      // 麦克风数据输入引脚
    };
    // 应用引脚配置
    i2s_set_pin(I2S_NUM_0, &pin_config_mic);
    // -----------------------------------------------------------
    // 2. 配置 I2S_NUM_1 用于扬声器 (播放/输出)
    // -----------------------------------------------------------
    i2s_config_t i2s_config_amp = {
      .mode = (i2s_mode_t)(I2S_MODE_MASTER | I2S_MODE_TX),   // 模式：主机(Master) + 发送(TX)    ESP32 产生时钟并将数据推送到扬声器
      // 以下参数必须与录音配置保持一致，否则声音会变调或失真
      .sample_rate = SAMPLE_RATE,
      .bits_per_sample = I2S_BITS_PER_SAMPLE_16BIT,
      .channel_format = I2S_CHANNEL_FMT_ONLY_LEFT,
      .communication_format = I2S_COMM_FORMAT_I2S,
      .intr_alloc_flags = ESP_INTR_FLAG_LEVEL1,
      .dma_buf_count = 8,
      .dma_buf_len = 64,
      .use_apll = false
    };
     // 安装驱动：初始化 I2S_NUM_1 硬件
    i2s_driver_install(I2S_NUM_1, &i2s_config_amp, 0, NULL);
    // 引脚映射配置
    i2s_pin_config_t pin_config_amp = {
      .bck_io_num = AMP_I2S_BCLK,
      .ws_io_num = AMP_I2S_LRC,
      .data_out_num = AMP_I2S_DIN,   // 扬声器数据输出引脚
      .data_in_num = -1              // 扬声器不需要输入引脚
    };
    // 应用引脚配置
    i2s_set_pin(I2S_NUM_1, &pin_config_amp);
  }
  ```

# 第二部分：后端 (Django) —— “仓库管理员”

### 1. 原理讲解

Django 是整个系统的大脑和仓库。它负责两件事：

1. **收货**：接收 ESP32 发来的文件，保存到硬盘，并在账本（数据库）上记一笔。
2. **发货**：当前端网页来询问时，把账本（录音列表）给它看。

#### 关键技术点：

- **路由 (URL Routing)**：就像门牌号。`/api/upload/` 是收货的门，`/api/history/` 是查询的门。
- **ORM (对象关系映射)**：你不需要写复杂的 SQL 语句（如 `INSERT INTO...`）。你只需要定义一个 Python 类 (`SensorData`)，Django 就会自动帮你管理数据库表。
- **Media Root (媒体库)**：Django 默认只处理代码文件。对于用户上传的图片、音频，我们需要指定一个专门的文件夹 (`media/`) 来存放。
- **静态服务**：默认情况下，出于安全，Django 不会让外部直接访问文件。我们需要特别配置，允许通过 URL（如 `http://.../media/audio.wav`）直接下载文件。

### 2. 核心配置与代码对应

- **配置仓库位置 (`core/settings.py`)**：

  Python

  ```
  MEDIA_ROOT = os.path.join(BASE_DIR, 'media') // 文件实际存哪 (硬盘路径)
  MEDIA_URL = '/media/'                        // 网页怎么访问 (网址前缀)
  ALLOWED_HOSTS = ['*']                        // 允许任何人(ESP32/手机)访问
  ```

- **定义账本 (`device_api/models.py`)**：

  Python

  ```
  class SensorData(models.Model):
      // FileField 会自动帮你处理文件保存逻辑
      audio_file = models.FileField(upload_to='audio/', ...) 
      timestamp = models.DateTimeField(auto_now_add=True)
  ```

- **处理收发货 (`device_api/views.py`)**：

  - **接收 (`receive_data`)**： `audio = request.FILES.get('audio_file')` -> 从请求中拿出文件。 `SensorData.objects.create(audio_file=audio)` -> 存硬盘 + 写数据库，一步到位。
  - **查询 (`get_history`)**： `request.build_absolute_uri(...)` -> 把相对路径 `/media/abc.wav` 变成完整网址 `http://172.xx.xx.xx:8000/media/abc.wav`，否则前端找不到。

# 第三部分：前端 (React) —— “展示柜台”

### 1. 原理讲解

React 是用户看到的界面。它本身不存数据，它只是数据的搬运工和展示者。

#### 关键技术点：

- **轮询 (Polling)**：因为 HTTP 是“请求-应答”模式，服务器不能主动推消息给前端。所以前端需要每隔几秒（`setInterval`）问一次服务器：“有新录音吗？”
- **状态管理 (State)**：`useState` 用来存从服务器拿到的数据列表。一旦数据变了，React 会自动刷新界面。
- **HTML5 Audio**：浏览器自带播放音频的能力。我们只需要把后端生成的 URL 喂给 `<audio src="...">` 标签，浏览器就会自己加载并播放。

### 2. 核心配置与代码对应

**文件：`frontend/src/App.jsx`**

- **定时去问服务器 (`useEffect` + `setInterval`)**：

  JavaScript

  ```
  useEffect(() => {
      fetchData() // 第一次加载
      const interval = setInterval(fetchData, 2000) // 每2秒问一次
      // ...
  }, [])
  ```

- **拿数据 (`fetchData`)**： 利用 Axios 发送 GET 请求到 `http://172.xx.xx.xx:8000/api/history/`。

- **自动播放逻辑**： 这是一个小技巧。我们记录上一次播放的 ID (`lastPlayedIdRef`)。如果新拿到的数据里，最新的 ID 变了，说明有新录音，于是触发 `audioRef.current.play()`。

# 总结：一次完整的“数据旅行”

1. **你按下按键**：ESP32 开始将麦克风声音写入 PSRAM。
2. **你松开按键**：
   - ESP32 给数据加上 WAV 头。
   - ESP32 连上 WiFi，找到电脑 IP。
   - ESP32 发起 HTTP POST 请求，把 WAV 数据“扔”给 Django。
3. **Django 接住数据**：
   - 把文件保存在 `media/audio/` 文件夹下。
   - 在 `db.sqlite3` 数据库里写下一行：“某年某月某日，存了一个文件，路径是...”
4. **React (2秒后)**：
   - 问 Django：“有新消息吗？”
   - Django 说：“有！最新的是这个...”
   - React 拿到 URL，更新列表，并自动开始播放。

这就是你亲手搭建的物联网系统的全部原理！理解了这些，你就真正掌握了这个项目。
